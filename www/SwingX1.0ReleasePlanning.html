Infrastructure

Filtering/Sorting

  • consistent support/api for all "Collection Views"
  • sort across several columns - supported in pipeline
  • provide "slim" hooks for custom implementations (like GlazedLists, JGA), implied: hide implementation details (f.i. FilterPipeline)
  • Performance: "fine-grained" update notification/handling
  • enable smooth migration to Mustang
  • (?) shareable across views - this is mandatory for use in JNTable with rowHeaders enabled (in the current implementation)

Searching

  • consistent support/api for all "Collection Views" - and others where applicable
  • UI-specification
  • add more search-widgets (QuickSearch,...)

Autocomplete

  • use internally - candidates?
  • (?)  add preliminary JXComboBox with auto-complete?

LookAndFeel

  • per-component LF addons


Context-dependent Popup Support

  • default popup - OS dependent?
  • implement getPopupLocation(...) to return something reasonable
  • (?) enhance ComponentPopupMenu to enable context-dependent popups
  • (?) enhance PopupEvent to include source-event (or other info) useful for a ContextProvider
  • (?) add pluggable ContextProvider

I18n

  • get all resources from UIManager instead of hardcoding
  • provide and use default framework resourceBundle: currently <swingx-path>/plaf/resources/swingx.properties - this is the default resourcebundle loaded in LFAddons. Open: guarantee loading even without a dummy XXAddOn?
  • make component R-to-L (component-orientation) aware
